Debugging¶

The following are ways to debug issues you might run into when working on DFFML.

Many times issues are because a package you are working on is not installed in development mode. First try checking ~/.local/lib/python3.7/site-packages/ and if you see anything other than .egg-link behind the name of the dffml package you are working on, you probably need to delete that package and re-install it in development mode (pip install -e).

Pop open debugger on execption¶

Run with pdb and input c to start execution.

  • w to print stack trace.

  • print(locals()) to print all local variables.

$ python -m pdb -m dffml --help
(Pdb) c
Traceback ...
(Pdb) w
(Pdb) print(locals())

Plugin Loading / Entrypoint Issues¶

If you can’t load the plugin you’ve been working on via the command line interface, HTTP API, etc. It’s probably an entry point issue.

The dev service has a helper command to help you debug issues with installed entrypoints.
